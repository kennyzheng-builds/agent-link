# A2A 项目决策记录

## 核心原则（第一性原理）

### 为什么做这个项目？
> "我的 Agent 有一个问题解决不了，只能问其他人，其他人再问他的 Agent，人类在中间传达，效率很低，而且人类的文字容易信息折损。"

**核心价值**：让 Agent 之间直接对话，消除人类传话的信息损耗。

### 安全模型
> "在目前的 Agent 早期，人来保证 Agent 的安全，以及知道哪个 Agent 能帮你解决。"

- **人工牵线**：人类决定找谁帮忙（基于对人的了解）
- **Agent 自主对话**：一旦建立连接，Agent 直接交流
- **人类确认收尾**：关键决策点需要主人批准

## 关键决策

### 1. 协作模式：老板-小弟模型
```
老板 A ←────确认合作────→ 老板 B
   ↓                       ↓
小弟 a ←────具体执行────→ 小弟 b
   ↑                       ↑
随时汇报               随时汇报
```

- 老板拍板：建联、关键决策、关闭会话
- 小弟执行：技术细节沟通、方案尝试
- 双向同步：任何一方都可以随时向主人汇报

### 2. 连接机制
| 方案 | 决策 | 理由 |
|-----|------|------|
| P2P vs 中继 | **轻量中继** | NAT 穿透简单，MVP 成本低 |
| 连接方式 | **连接码/短链接** | 像 Zoom 会议号一样直观 |
| 身份绑定 | **Google 邮箱** | 已有基础设施，用户熟悉 |

### 3. 隐私与开源
| 阶段 | 策略 |
|-----|------|
| MVP | 点到点加密，内容存本地，不公开 |
| Phase 2 | 主人授权后，可脱敏分享经验到社区 |
| 长期 | Agent 经验开源，类似 GitHub for Agents |

### 4. 成本策略
- **目标**：MVP 阶段月成本 <$10，最好免费
- **选型**：Cloudflare Workers（免费额度够用）
- **演进**：从中心化中继逐步过渡到 P2P

## 术语定义

| 术语 | 含义 |
|-----|------|
| **主人** | 拥有 OpenClaw Agent 的人类用户 |
| **Agent** | 主人的 AI 助手（OpenClaw 实例） |
| **连接码** | 6位字母数字，用于建立会话（如 A3F9K2） |
| **中继** | 协调连接的服务器，不存储消息内容 |
| **会话** | 两个 Agent 之间的临时通信通道 |
