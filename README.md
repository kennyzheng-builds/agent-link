# Agent Link

> 让 Agent（而非人类）来打包和解读问题上下文，消除人类传话的信息损耗。

## 核心问题

你和朋友各有一个 AI Agent。你遇到技术问题，朋友的 Agent 擅长解决。但人类在中间传话时，报错信息、代码片段、配置细节会大量丢失。Agent 只要有完整上下文，大多数问题都能解决——**瓶颈在人类传话造成的信息丢失**。

## 怎么用

```
你遇到问题，自己的 Agent 解决不了
    ↓
你："帮我把这个问题打包，我找老王帮忙看看"
    ↓
Agent 收集上下文（报错、代码、已尝试方案），生成「协作请求」
    ↓
你复制这段文本，微信/飞书发给老王
    ↓
老王粘贴给自己的 Agent → Agent 识别、分析、生成「协作回复」
    ↓
老王把回复发回给你 → 你的 Agent 读取并告诉你下一步怎么做
```

## 特点

- **信息零损耗**：Agent 打包完整上下文，不再依赖人类转述
- **零成本**：纯 Skill 文件，不需要服务器、域名、数据库
- **安全**：只读咨询模式，自动过滤 API Key 等敏感信息
- **人类可读**：协作请求/回复是标准 Markdown，即使不用 Agent 也能直接看

## 安装
